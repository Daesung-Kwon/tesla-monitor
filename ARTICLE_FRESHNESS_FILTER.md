# 📅 기사 최신성 필터 추가!

오래된 기사가 알림으로 오는 문제를 해결했습니다!

---

## ❌ 이전 문제

### 현상:
```
📱 Telegram 알림:
- "Tesla Model Y 2020 Review" (4년 전 기사)
- "Cybertruck Announcement 2019" (5년 전 기사)
- "Tesla Stock Split 2022" (2년 전 기사)
```

### 원인:
```python
# 기존 로직
for entry in feed.entries[:10]:
    if article_id not in seen_articles:
        # 날짜 체크 없이 모두 "새 기사"로 간주 ❌
        send_notification(entry)
```

**문제점:**
- ❌ 첫 실행 시 `seen_articles`가 비어있음
- ❌ RSS 피드의 모든 기사를 "새 기사"로 간주
- ❌ 2020년, 2019년 기사도 알림으로 옴
- ❌ 날짜 필터링 없음

---

## ✅ 해결 방법

### 추가된 로직:

```python
# 1. 기사 날짜 파싱
pub_date = parse_date(entry.published)

# 2. 현재 시간과 비교
age_days = (now - pub_date).days

# 3. 최근 N일 이내만 처리 (기본 3일)
if age_days <= MAX_ARTICLE_AGE_DAYS:
    send_notification(entry)  ✅
else:
    skip(entry)  # 오래된 기사 스킵
```

## 🔧 설정 방법

### GitHub Secrets에 추가 (선택 사항)

```
Name: MAX_ARTICLE_AGE_DAYS
Secret: 3
```

**설정하지 않으면:** 기본값 3일 사용

### 권장 값:

| 값 | 의미 | 추천 대상 |
|----|------|----------|
| **1** | 최근 1일 | 매우 최신만 원함 |
| **3** | 최근 3일 | **추천 (기본값)** |
| **7** | 최근 1주일 | 주 1회 체크 |
| **14** | 최근 2주일 | 많은 알림 원함 |

## 💡 날짜 파싱 로직

### 다양한 RSS 날짜 형식 지원:

```python
# RFC 822 (대부분의 RSS)
"Sat, 18 Jan 2026 10:30:00 GMT"

# RFC 3339 (Atom)
"2026-01-18T10:30:00Z"

# 다양한 시간대
"Sat, 18 Jan 2026 10:30:00 PST"
"Sat, 18 Jan 2026 10:30:00 +0900"
```

**자동 처리:**
- ✅ 시간대 자동 인식
- ✅ UTC로 변환
- ✅ 현재 시간과 비교

**에러 처리:**
- 날짜 파싱 실패 시 → 최신 기사로 간주 (보수적 접근)
- 날짜 정보 없음 → 최신 기사로 간주
